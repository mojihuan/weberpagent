## 综合测试用例设计 (per D-01)

### 用例名称: 采购入库全流程验证
### 场景描述: 创建一个采购入库相关测试任务，包含前置条件（查询供应商信息）和业务断言（验证入库数量），确保触发 AI 推理生成 Eval/Verdict/Memory/Goal

### 前置条件配置:
- 添加 1 个前置条件：查询供应商列表（使用数据获取方法，如 `context.get_data('SupplierList', 'query', {})`）
- 设置变量名如 `supplier_name` 用于后续引用

### 业务断言配置:
- 添加 1 个业务断言：验证库存数量（使用断言方法，headers='main'）
- 配置字段参数验证某个数量字段

### 任务描述模板:
"进入采购管理模块，点击新建采购入库单，选择供应商 {{supplier_name}}，填写入库数量为 1，提交并验证入库成功"

## SC-1: 执行监控步骤验证 (per D-06-1)

**验证目标:** 创建含前置条件+断言的测试任务，执行后监控中正确显示所有步骤（含推理格式化）

### 操作步骤:
1. 打开平台首页 http://localhost:11002
2. 点击"新建任务"按钮
3. 填写任务名称: "v0.8.0-采购入库全流程验证"
4. 填写任务描述（使用上述模板）
5. 配置前置条件（添加查询供应商信息代码）
6. 配置业务断言（添加库存验证断言）
7. 点击"创建"提交任务
8. 点击"执行"开始执行
9. **观察执行监控页面**

### 验证条件:
- [ ] StepTimeline 中显示前置条件步骤（per EXEC-01）：显示状态（成功/失败）、耗时、代码摘要
- [ ] StepTimeline 中显示断言步骤（per EXEC-02）：显示状态（通过/失败）、耗时、断言名称
- [ ] 前置条件步骤、断言步骤与普通 UI 步骤按执行顺序交错排列（per EXEC-03）
- [ ] ReasoningLog 中推理文本按行格式化展示（per FMT-01）：Eval/Verdict/Memory/Goal 各自独占一行
- [ ] 推理标签有视觉高亮（颜色标记）
