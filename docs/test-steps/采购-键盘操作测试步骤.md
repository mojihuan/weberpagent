# 商品采购模块键盘操作测试步骤

## TestPurchaseKeyboard - 采购管理 | 键盘操作

### test_keyboard_enter_search - [键盘]-采购单物品编号回车搜索

前置条件：已登录 ERP 系统

步骤1：{点击}商品采购
步骤2：{点击}采购管理
步骤3：{点击}新增采购单
步骤4：{输入}IMEI/物品编号{输入物品编号}
步骤5：{send_keys}物品编号输入框{Enter}

预期：Agent 使用 send_keys('Enter') 在物品编号输入框触发搜索，搜索结果列表出现匹配项

验证条件：
- 成功：Agent 在输入物品编号后主动使用 send_keys('Enter') 触发搜索，搜索下拉出现匹配结果
- 失败：Agent 未使用 send_keys('Enter')，而是点击搜索按钮或其他方式触发搜索

断言：调用方法：pc.purchase_order_list_assert(purchaseTime='now', stateStr='已发货')

---

### test_keyboard_escape_close_popup - [键盘]-ESC关闭日期选择器

前置条件：采购单表单已打开

步骤1：{点击}日期字段触发日期选择器弹窗
步骤2：{send_keys}日期选择器{Escape}

预期：Agent 使用 send_keys('Escape') 关闭日期选择器弹窗，回到表单状态

验证条件：
- 成功：Agent 在日期选择器弹出后主动使用 send_keys('Escape') 关闭弹窗，弹窗消失回到表单
- 失败：Agent 未使用 send_keys('Escape')，而是点击弹窗外区域或点击关闭按钮

断言：调用方法：pc.purchase_order_list_assert(purchaseTime='now')

---

### test_keyboard_select_all_overwrite - [键盘]-全选覆盖输入框内容

前置条件：采购单表单已打开，有含内容的输入框

步骤1：{点击}备注输入框
步骤2：{send_keys}备注输入框{Control+a}
步骤3：{输入}备注输入框{新备注内容}

预期：Agent 使用 send_keys('Control+a') 全选输入框内容，再用 input 输入新值覆盖，输入框显示新值而非旧值

验证条件：
- 成功：Agent 先使用 send_keys('Control+a') 全选原有内容，再 input 新值覆盖，输入框显示新备注内容
- 失败：Agent 未使用 send_keys('Control+a')，而是逐字符删除或直接输入导致内容追加

断言：调用方法：pc.purchase_order_list_assert(purchaseTime='now')
