# 商品采购模块表格交互测试步骤

## TestPurchaseTableInteraction - 采购管理 | 表格交互

### test_table_checkbox_row_select - [表格]-采购单列表单行 checkbox 选择

前置条件：已登录 ERP 系统，采购单列表有数据

步骤1：{点击}商品采购
步骤2：{点击}采购管理
步骤3：{点击}采购单列表中某一行 tbody 中的 checkbox

预期：Agent 通过 DOM 位置定位 tbody 行中的 checkbox 并点击，该行被选中（checkbox 勾选状态），其他行未选中

验证条件：
- 成功：Agent 定位 tbody 中的 checkbox 元素并点击，仅该行被选中
- 失败：Agent 通过索引或坐标定位 checkbox，或点击了 thead 中的全选 checkbox

Agent 动作说明：click tbody 行中的 checkbox，通过 DOM 层级位置（tbody）区分行选择 checkbox 和表头全选 checkbox

断言：调用方法：pc.purchase_order_list_assert(purchaseTime='now')

---

### test_table_checkbox_select_all - [表格]-采购单列表表头全选 checkbox

前置条件：已登录 ERP 系统，采购单列表有多行数据

步骤1：{点击}商品采购
步骤2：{点击}采购管理
步骤3：{点击}表头 thead 中的全选 checkbox

预期：Agent 通过 DOM 位置定位 thead 中的全选 checkbox 并点击，列表所有行均被选中

验证条件：
- 成功：Agent 定位 thead 中的 checkbox 元素并点击，所有行被选中
- 失败：Agent 点击了 tbody 行中的 checkbox（单选而非全选），或通过索引/坐标定位

Agent 动作说明：click thead 中的 checkbox，通过 DOM 层级位置（thead）区分全选 checkbox 和行选择 checkbox

断言：调用方法：pc.purchase_order_list_assert(purchaseTime='now')

---

### test_table_link_click - [表格]-采购单列表超链接点击

前置条件：已登录 ERP 系统，采购单列表有数据

步骤1：{点击}商品采购
步骤2：{点击}采购管理
步骤3：{点击}某个采购单号超链接文本

预期：Agent 通过可见文本直接 click 超链接（<a> 元素），页面跳转到该采购单详情页或弹出详情弹窗

验证条件：
- 成功：Agent 通过可见文本匹配直接点击 <a> 超链接元素，页面跳转或弹出详情
- 失败：Agent 使用 find_elements 预查找，或通过索引/坐标定位超链接

Agent 动作说明：click 可见文本匹配的 <a> 元素，无需 find_elements 预查找

断言：调用方法：pc.purchase_order_list_assert(purchaseTime='now')

### test_table_icon_button_click - [表格]-采购单列表操作按钮点击

前置条件：已登录 ERP 系统，采购单列表有数据

步骤1：{点击}商品采购
步骤2：{点击}采购管理
步骤3：{点击}某行的"编辑"或"查看"图标按钮

预期：Agent 通过 title 或 aria-label 属性定位图标按钮并点击，进入编辑页面或弹出详情弹窗

验证条件：
- 成功：Agent 通过 title 或 aria-label 属性匹配定位图标按钮并点击，页面跳转或弹窗打开
- 失败：Agent 通过索引或坐标定位按钮，或无法找到正确的图标按钮

Agent 动作说明：click title 或 aria-label 属性匹配的元素，定位行末操作按钮

断言：调用方法：pc.purchase_order_list_assert(purchaseTime='now')
