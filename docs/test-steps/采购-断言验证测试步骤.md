### test_assertion_headers_param - [断言]-headers 参数传递验证 (AST-01)

前置条件：已登录 ERP 系统，WEBSERP_PATH 已配置

步骤1：{点击}商品采购
步骤2：{点击}采购管理

断言配置：
- class_name: PcAssert
- method_name: purchase_order_list_assert
- headers: 'main'
- data: 'main'

预期：断言调用成功执行（非 ImportError/HeaderResolutionError），说明 headers='main' 参数被正确传递到 execute_assertion_method 并通过 _process_headers 解析为实际 headers dict

验证条件：
- 成功：断言执行结果 success=True（无 HeaderResolutionError），返回 passed=True 或 AssertionError（业务断言结果）
- 失败：error_type='HeaderResolutionError' 或 'ImportError'

Agent 动作说明：登录 ERP，进入采购管理页面，平台自动调用断言接口验证数据

注意事项：
- headers 参数标识符可选值: main, idle, vice, special, platform, super, camera
- data 参数选项: main=主数据, a=配件数据, b=分店数据, c=仓库数据, d=其他数据
- 本场景使用 headers='main' 验证参数传递链路是否通畅

断言：调用方法：pc.purchase_order_list_assert(headers='main', data='main')

### test_assertion_ij_params - [断言]-inventory_list_data i/j 参数组合验证 (AST-02)

前置条件：已登录 ERP 系统，WEBSERP_PATH 已配置，库存数据存在

步骤1：{点击}商品采购
步骤2：{点击}采购管理

断言配置：
- class_name: PcAssert
- method_name: inventory_list_data
- data: 'main'
- api_params: {"i": "2", "j": "13"}

预期：i/j 参数正确传递到 API 调用，返回有效的库存列表数据（非空列表）

验证条件：
- 成功：断言调用返回数据且包含库存记录，success=True
- 失败：error_type='ParameterError' 或返回数据为空

Agent 动作说明：登录 ERP，进入库存管理页面，平台自动调用断言接口验证 i/j 参数组合

注意事项：
- i（库存状态）: 1=待入库, 2=库存中, 3=已出库
- j（物品状态）: 3=待分货, 13=待销售
- 本场景使用 i=2（库存中）+ j=13（待销售）组合，验证参数是否正确传递到 API 调用
- api_params 中的 i/j 值会通过 execute_assertion_method 的 merged_kwargs 传递到方法调用

断言：调用方法：pc.inventory_list_data(data='main', i=2, j=13)
