# 设计文档：前置条件自包含代码生成

**日期**: 2026-04-30
**状态**: 已确认

## 问题

生成的 Playwright 测试代码中，前置条件产生的动态数据（如 IMEI）被硬编码。
当 ERP 数据状态变化（物品已出库）后，生成的代码无法通过测试。

当前生成代码：
```python
page.locator('input[placeholder="请输入IMEI"]').fill("I01778194069604", timeout=5000)
```
应该生成：
```python
imei = random_imei()
...
page.locator('input[placeholder="请输入IMEI"]').fill(imei, timeout=5000)
```

## 目标

生成的 `.py` 文件完整自包含，可直接 `pytest` 运行。前置条件代码嵌入生成文件，
动态数据每次运行重新生成。

## 方案：前置条件代码嵌入 + Jinja2 替换记录追踪

### 数据流变更

```
前置条件执行 → context 变量 → Jinja2 替换任务描述 → agent 操作 → fill() 代码
                  ↓                        ↓
          variable_map          substitution_map 记录
          {imei: "I01778..."}   {原始文本→替换后文本}
                  ↓                        ↓
              传递到 StepCodeBuffer → 反向查找：fill 值 → 变量名 → 替换为变量引用
```

### 核心数据结构

```python
variable_map: dict[str, str]  # 变量名 → 实际值，如 {"imei": "I01778194069604"}
```

映射已存在于 `PreconditionService.context.to_dict()`，只需传递到代码生成阶段。

### 代码变更

#### 1. `code_generator.py` — 前置条件代码注入

- `_build_precondition()` 方法扩展：接收原始前置条件代码 `precondition_code: list[str]`
- 在 `page.goto()` 之前注入随机生成函数定义 + 前置条件代码
- 新增 `_substitute_variables_in_code()` 用 variable_map 反向替换 fill 中的硬编码值

#### 2. `StepCodeBuffer` — 传递变量映射

- `assemble()` 方法新增 `variable_map` 参数
- 传递给 `PlaywrightCodeGenerator.generate()`

#### 3. `run_pipeline.py` — 串联数据

- 前置条件执行后，`context.to_dict()` 已有变量映射
- 将映射和原始前置条件代码传递给 `assemble()`

#### 4. 值替换逻辑

```python
def _substitute_variables_in_code(code: str, variable_map: dict[str, str]) -> str:
    for var_name, actual_value in variable_map.items():
        if actual_value in code:
            code = code.replace(f'"{actual_value}"', var_name)
    return code
```

### 边界情况处理

| 场景 | 处理方式 |
|------|---------|
| `context.get_data()` 外部调用 | 注释 + 硬编码 fallback |
| 变量值是数字或特殊字符 | 只替换字符串值，且只在引号内替换 |
| 前置条件代码有语法错误 | 降级为不嵌入（保持现有行为） |
| 无前置条件 | 不注入，保持现有行为 |
| `context.cache()` 依赖 | 不支持，降级为硬编码 |

### 随机函数处理

内联定义到生成的 `.py` 文件中（`random_imei`, `random_phone` 等），
不依赖外部模块，确保完全自包含。

### 生成文件示例

```python
# Generated by aiDriveUITest
# Task: 入库扫描
# ...

from playwright.sync_api import Page
import random


def random_imei():
    return "I" + ''.join(random.choices('0123456789', k=14))


def test_入库扫描(page: Page) -> None:
    """Auto-generated test from agent execution"""
    # === Precondition: 动态数据生成 ===
    imei = random_imei()

    page.goto("http://erp.example.com")
    try:
        page.wait_for_load_state("networkidle", timeout=10000)
    except Exception:
        pass

    page.get_by_role("button", name="入库管理").click()
    ...
    page.locator('input[placeholder="请输入IMEI"]').fill(imei, timeout=5000)
    ...
```

### 外部调用示例（get_data fallback）

```python
def test_入库扫描(page: Page) -> None:
    # === Precondition: 动态数据生成 ===
    imei = random_imei()
    # NOTE: context.get_data('BaseParams', 'inventory_list_data', i=2, j=13)
    #       外部调用无法自包含，使用执行时的值作为 fallback
    warehouse = "华东仓"

    page.goto("http://erp.example.com")
    ...
```
