# 发货单表单填写测试设计

## 1. 背景

### 目标

验证 SimpleAgent 能否完成 ERP 系统中复杂表单（发货单）的填写任务。

### 验证能力

- 多级菜单导航能力
- 弹窗表单操作能力
- 复杂表单填写能力（基础字段 + 选择器 + 明细表格）
- 完整流程验证能力

---

## 2. 测试流程

```
┌─────────────────────────────────────────────────────────────┐
│                      完整测试流程                             │
├─────────────────────────────────────────────────────────────┤
```

## 3. 技术设计

### 3.1 测试文件

**路径**：`backend/tests/test_delivery_form.py`

### 3.2 核心组件

#### 随机数据生成器

```python
def generate_random_delivery_data():
    """生成随机发货单数据"""
    import random
    import string

    suffix = ''.join(random.choices(string.ascii_lowercase, k=6))
    return {
        'receiver': f'测试收货人_{suffix}',
        'phone': f'1{random.choice("3456789")}{random.randint(100000000, 999999999)}',
        'address': f'测试地址_{suffix}号',
    }
```

#### 测试用例

```python
@pytest.mark.asyncio
async def test_delivery_form_fill():
    """测试发货单表单填写完整流程"""

    # 1. 生成随机测试数据
    test_data = generate_random_delivery_data()

    # 2. 创建 Agent
    agent = SimpleAgent(
        task=f"""
        执行发货单填写任务：
        1. 登录 ERP（用户名：Y59800075，密码：Aa123456）
        2. 导航到：商品采购 → 采购管理 → 新增发货单
        3. 点击"+新增"打开表单
        4. 填写发货单信息：
           - 收货人：{test_data['receiver']}
           - 电话：{test_data['phone']}
           - 地址：{test_data['address']}

        5. 保存发货单

        """,
```

## 4. 验收标准

| 指标 | 期望 | 说明 |
|------|------|------|
| 完成率 | ≥ 80% | Agent 能完成大部分步骤 |
| 提交成功 | ✅ | 表单能成功提交（无验证错误） |
| 数据验证 | ✅ | 能在列表中找到新创建的发货单 |
| 截图覆盖 | 100% | 每步都有截图记录 |
| 执行时间 | ≤ 5 分钟 | 完整流程执行时间 |

---

## 5. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 元素定位失败 | 动作无法执行 | Agent 反思机制自动重试 |
| 表单验证失败 | 提交被拒绝 | 生成符合格式的随机数据 |
| 页面加载慢 | 超时 | 增加 max_steps 和等待策略 |
| 网络问题 | 连接失败 | 支持重试机制 |
