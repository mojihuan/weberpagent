## 2. 采购单场景复杂度分析

### 2.1 任务流程拆解

采购单场景包含 **12 个子任务**：

```
1. 打开首页
2. 输入账号
3. 输入密码
4. 点击登录
5. 点击侧边栏"采购管理"
6. 点击"采购订单"
7. 点击"商品采购"
8. 点击"新增"按钮
9. 选择设备类型（下拉框）
10. 填写表单必填字段（多个输入框）
11. 点击提交/保存
12. 确认成功
```

### 2.2 复杂度对比

| 维度 | 登录场景 | 采购单场景 | 倍数 |
|------|----------|------------|------|
| 子任务数 | 5 | 12 | 2.4x |
| 页面导航 | 1 | 4 | 4x |
| 表单交互 | 2 | 5+ | 2.5x |
| 动态内容 | 低 | 高 | - |

### 2.3 核心难点

#### 难点 1: 多级侧边栏导航

```
采购管理 → 采购订单 → 商品采购
```

- 需要连续 3 次精准点击
- 每次点击后需要等待子菜单展开
- 菜单可能需要悬停才能显示

#### 难点 2: 复杂表单控件

采购单表单可能包含：
- **下拉选择框**: 设备类型选择（需要点击展开 + 选择选项）
- **日期选择器**: 可能需要特殊处理
- **动态加载字段**: 某些字段可能根据前序选择动态出现
- **必填校验**: 需要识别哪些字段是必填的

#### 难点 3: 成功验证

- 提交后需要检测"新记录出现在列表中"
- 页面可能有延迟加载
- 成功提示可能是 toast 通知（短暂出现）

## 3. 失败原因分析

### 3.1 Phase 4 失败原因推测

基于 25 步仍未完成的现象，可能的失败点：

| 阶段 | 可能问题 | 概率 |
|------|----------|------|
| 侧边栏导航 | 无法识别多级菜单结构 | 高 |
| 侧边栏导航 | 点击后子菜单未展开 | 中 |
| 表单填写 | 无法定位下拉框选项 | 高 |
| 表单填写 | 必填字段识别错误 | 中 |
| 提交验证 | 无法判断提交成功 | 中 |

### 3.2 与登录场景的差异

登录场景在 Phase 5 成功的关键因素：
1. 正确识别需要切换到"密码登录"模式
2. 使用 placeholder 定位输入框
3. 正确判断登录成功（检测页面标题变化）

采购单场景需要额外的能力：
1. **多级菜单导航** - 登录场景不涉及
2. **下拉框操作** - 登录场景不涉及
3. **动态表单处理** - 登录场景表单固定

### 4.2 执行层优化

#### 4.2.1 增强下拉框处理

```python
async def _handle_select(
    self,
    target: str,
    value: str,
    elements: list[InteractiveElement],
) -> ActionResult:
    """处理下拉选择框"""
    # 1. 先点击展开下拉框
    locator = await self._locate_element(target, elements)
    if locator:
        await locator.click()

        # 2. 等待选项列表出现
        await self.page.wait_for_timeout(500)

        # 3. 点击目标选项
        option_locator = self.page.get_by_text(value)
        await option_locator.click(timeout=3000)

        return ActionResult(success=True)
```

#### 4.2.2 增强等待策略

```python
# 导航后增加等待时间
async def _navigate(self, url: str) -> ActionResult:
    await self.page.goto(url, wait_until="domcontentloaded")
```

```python
def _detect_navigation_failure(self) -> bool:
    """检测导航是否失败"""
    # 如果连续点击但 URL 未变化，可能是导航失败
    if len(self.history) >= 3:
        recent_urls = [s.state.url for s in self.history[-3:]]
        if len(set(recent_urls)) == 1:
            # URL 未变化，检查是否有菜单展开
            # 可能需要悬停而非点击
            return True
    return False
```

## 7. 结论

采购单场景失败的核心原因是：

1. **任务复杂度高**：12 个子任务 vs 登录场景的 5 个
2. **多级导航**：侧边栏 3 级菜单导航，Phase 5 未针对性优化
3. **复杂表单**：下拉框、动态字段等，登录场景不涉及

**建议采用渐进式优化策略**：
1. 先通过 Prompt 优化快速验证（低成本）
2. 效果不理想再深入执行层/感知层（高成本）
