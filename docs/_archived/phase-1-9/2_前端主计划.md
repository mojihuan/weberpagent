## 3. 后端 API 设计

### 3.1 路由设计

```python
# backend/api/main.py - FastAPI 路由设计

# ===== 任务管理 =====
GET    /api/tasks              # 获取任务列表
POST   /api/tasks              # 创建新任务
GET    /api/tasks/{id}         # 获取任务详情
PUT    /api/tasks/{id}         # 更新任务
DELETE /api/tasks/{id}         # 删除任务

# ===== 执行管理 =====
POST   /api/runs               # 启动执行（传入任务ID）
GET    /api/runs               # 获取执行历史列表
GET    /api/runs/{id}          # 获取单次执行详情（含步骤、截图）
POST   /api/runs/{id}/stop     # 停止执行
GET    /api/runs/{id}/stream   # SSE 实时推送执行进度

# ===== 报告查看 =====
GET    /api/reports            # 获取报告列表
GET    /api/reports/{id}       # 获取报告详情
GET    /api/screenshots/{path} # 获取截图文件

# ===== 配置管理 =====
GET    /api/config/targets     # 获取测试目标配置
PUT    /api/config/targets     # 更新测试目标配置
GET    /api/config/models      # 获取可用模型列表
```

### 3.2 数据结构

```typescript
// 任务
interface Task {
  id: string
  name: string
  description: string          // 自然语言任务描述
  target_url: string
  max_steps: number
  status: 'draft' | 'ready'
}

// 执行记录
interface Run {
  id: string
  task_id: string
  status: 'running' | 'success' | 'failed' | 'stopped'
  started_at: string
  finished_at?: string
  steps: Step[]
}

// 单步执行
interface Step {
  index: number
  action: string              // AI 输出的动作
  reasoning?: string          // AI 的推理过程
  screenshot: string          // 截图路径
  status: 'success' | 'failed'
  error?: string
  duration_ms: number
}
```

### 4.1 页面结构

```

├── 首页/仪表盘 (/)
│   ├── 最近执行记录（5条）
```
