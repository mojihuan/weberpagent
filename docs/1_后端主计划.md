# AI + Playwright UI 自动化测试 POC 后端主文档

## ⚠️ 技术方向变更 (2026-03-09)

### 变更背景

在 Phase 4 场景验证过程中，发现 **Browser-Use 框架与国产模型存在兼容性问题**：

| 模型 | 问题 | 详情 |
|------|------|------|
| Azure OpenAI (gpt-5.4) | 内容过滤器触发 | Browser-Use 系统提示词被检测为 jailbreak |
| DeepSeek | JSON Schema 不兼容 | 无法输出正确的 AgentOutput 格式 |
| 通义千问 | action 字段缺失 | 无法输出结构化 action 对象 |

### 变更决定

**放弃 Browser-Use，改为自己实现简化版 Agent**

### 新方案优势

1. **完全控制 Prompt** - 针对国产模型优化，避免触发内容过滤器
2. **简化输出格式** - 不需要复杂的 AgentOutput Schema
3. **更容易调试** - 每个环节都可控
4. **代码量小** - 预计 200-300 行核心代码

### 设计文档

详见：`docs/plans/2026-03-09-simple-agent-design.md`

## 1. 项目概述

| 项目 | 内容 |
|------|------|
| **项目名称** | AI + Playwright UI 自动化测试 POC |
| **目标** | 验证 "AI 决策 + Playwright 执行" 技术闭环可行性 |
| **技术栈** | Python + Browser-Use + 国内模型（通义/智谱/DeepSeek） |

## 2. 核心设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 实现方式 | **自研简化版 Agent** | Browser-Use 与国产模型不兼容 |
| 编程语言 | 全 Python | 简化架构，AI 生态丰富 |
| 测试目标 | 真实业务系统 | 验证实际场景效果 |
| POC 场景 | 登录 + 表单 + 搜索 | 覆盖典型业务流程 |
| LLM 选择 | DeepSeek / 通义千问 | 国产模型，API 稳定 |

```
开始
  │
  ▼
┌─────────────────┐
│  1. 启动任务     │  ← 自然语言描述任务目标

         ▼
┌─────────────────┐
│  2. 感知页面     │  ← Playwright 截图 + Accessibility Tree
│  获取当前状态    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  3. AI 决策     │  ← 国内模型（通义/智谱/DeepSeek）
│  生成下一步动作  │     输入：截图 + DOM + 任务 + 历史
└────────┬────────┘     输出：结构化动作
         │
         ▼
┌─────────────────┐
│  4. 执行动作     │  ← Playwright 执行
│  转换并调用 API  │     click / fill / select / goto...
└────────┬────────┘
         │
         ▼
    ┌────────┐
    │ 成功？  │
    └───┬────┘
        │
   ┌────┴────┐
   │         │
  是        否
   │         │
   │         ▼
   │   ┌─────────────────┐
   │   │  5. 反思重试     │  ← 重新截图 + AI 分析失败原因
   │   │  生成修复策略    │     最多重试 N 次
   │   └────────┬────────┘
   │            │
   └────────────┼──────────────────────┐
                │                      │
                ▼                      ▼
         ┌─────────────┐        ┌─────────────┐
         │ 任务完成？   │   否   │  继续循环    │
         └──────┬──────┘ ────── │  返回步骤 2  │
                │               └─────────────┘
           是   │
                ▼
         ┌─────────────┐
         │ 输出报告     │
         │ 结束         │
         └─────────────┘
```

## 5. 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 最大步数 | 20 | 单个任务最多执行 20 步 |
| 单步超时 | 30s | 单个动作执行超时时间 |
| 最大重试 | 3 | 失败后最多重试 3 次 |
| 截图保存 | 是 | 每步都保存截图 |

## 6. 自研简化版 Agent 设计要点

### 核心模块

| 模块 | 职责 | 复杂度 |
|------|------|--------|
| **页面感知** | 获取截图 + 提取可交互元素 | ⭐ |
| **LLM 决策** | 调用模型 + 解析输出 | ⭐⭐ |
| **动作执行** | Playwright click/input/navigate | ⭐ |
| **循环控制** | 步数限制 + 完成检测 | ⭐ |

### 简化的输出格式

```json
{
  "thought": "分析当前页面状态...",
  "action": "click",
  "target": "密码登录",
  "value": null
}
```

相比 Browser-Use 的 AgentOutput Schema，这个格式更简单，国产模型更容易正确输出。

### 代码结构

```
backend/agent_simple/
├── __init__.py
├── agent.py          # 主循环逻辑
├── perception.py     # 页面感知
├── decision.py       # LLM 决策
├── executor.py       # 动作执行
└── prompts.py        # Prompt 模板
```

详见：`docs/plans/2026-03-09-simple-agent-design.md`

## 7. POC 验收标准

### 场景验收

| 场景 | 成功标准 | 验证方式 |
|------|----------|----------|
| **登录** | AI 能识别输入框 → 填写账号密码 → 点击登录 → 判断登录成功 | 截图 + 日志 + 断言 |
| **表单提交** | AI 能识别各类型输入 → 正确填写 → 提交 → 验证成功提示 | 截图 + 日志 + 断言 |
| **搜索** | AI 能输入关键词 → 点击搜索 → 识别搜索结果 | 截图 + 日志 + 断言 |

### 成功指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 场景通过率 | ≥ 80% | 3 个场景至少 2 个完全通过 |
| 单步推理耗时 | ≤ 10s | AI 单次决策时间 |
| 自愈成功率 | ≥ 50% | 失败后能通过重试恢复 |
| 截图完整率 | 100% | 每步都有截图记录 |
