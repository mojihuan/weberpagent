## 1. 项目概述

| 项目 | 内容 |
|------|------|
| **项目名称** | AI + Playwright UI 自动化测试 POC |
| **目标** | 验证 "AI 决策 + Playwright 执行" 技术闭环可行性 |
| **技术栈** | Python + Browser-Use + 国内模型（通义/智谱/DeepSeek） |

## 2. 核心设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 实现方式 | 基于 Browser-Use 改造 | 站在成熟框架上，专注模型适配 |
| 编程语言 | 全 Python | 简化架构，AI 生态丰富 |
| 测试目标 | 真实业务系统 | 验证实际场景效果 |
| POC 场景 | 登录 + 表单 + 搜索 | 覆盖典型业务流程 |

## 4. 核心执行流程

```
开始
  │
  ▼
┌─────────────────┐
│  1. 启动任务     │  ← 自然语言描述任务目标
│  初始化上下文    │

         │
         ▼
┌─────────────────┐
│  2. 感知页面     │  ← Playwright 截图 + Accessibility Tree
│  获取当前状态    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  3. AI 决策     │  ← 国内模型（通义/智谱/DeepSeek）
│  生成下一步动作  │     输入：截图 + DOM + 任务 + 历史
└────────┬────────┘     输出：结构化动作
         │
         ▼
┌─────────────────┐
│  4. 执行动作     │  ← Playwright 执行
│  转换并调用 API  │     click / fill / select / goto...
└────────┬────────┘
         │
         ▼
    ┌────────┐
    │ 成功？  │
    └───┬────┘
        │
   ┌────┴────┐
   │         │
  是        否
   │         │
   │         ▼
   │   ┌─────────────────┐
   │   │  5. 反思重试     │  ← 重新截图 + AI 分析失败原因
   │   │  生成修复策略    │     最多重试 N 次
   │   └────────┬────────┘
   │            │
   └────────────┼──────────────────────┐
                │                      │
                ▼                      ▼
         ┌─────────────┐        ┌─────────────┐
         │ 任务完成？   │   否   │  继续循环    │
         └──────┬──────┘ ────── │  返回步骤 2  │
                │               └─────────────┘
           是   │
                ▼
         ┌─────────────┐
         │ 输出报告     │
         │ 结束         │
         └─────────────┘
```

## 5. 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 最大步数 | 20 | 单个任务最多执行 20 步 |
| 单步超时 | 30s | 单个动作执行超时时间 |
| 最大重试 | 3 | 失败后最多重试 3 次 |
| 截图保存 | 是 | 每步都保存截图 |

## 6. Browser-Use 改造要点

### 需要改造的模块

| 模块 | 改造内容 | 优先级 |
|------|----------|--------|
| **LLM 适配层** | 新增国内模型（通义/智谱/DeepSeek）的调用适配 | P0 |
| **Prompt 模板** | 调整为中英文混合，优化中文场景理解 | P0 |
| **动作格式** | 保持 Browser-Use 原生格式，无需改造 | - |
| **日志输出** | 增加中文日志，便于调试 | P1 |
| **截图保存** | 确保每步截图正确保存到 outputs 目录 | P1 |

### 国内模型适配接口

```python
# backend/llm/base.py
from abc import ABC, abstractmethod
from typing import Any

class BaseLLM(ABC):
    """国内模型统一接口"""

    @abstractmethod
    async def chat_with_vision(
        self,
        messages: list[dict],
        images: list[str]  # base64 或 URL
    ) -> dict:
        """带图像理解的对话"""
        pass

    @abstractmethod
    def parse_action(self, response: str) -> dict:
        """解析模型输出为结构化动作"""
        pass
```

## 7. POC 验收标准

### 场景验收

| 场景 | 成功标准 | 验证方式 |
|------|----------|----------|
| **登录** | AI 能识别输入框 → 填写账号密码 → 点击登录 → 判断登录成功 | 截图 + 日志 + 断言 |
| **表单提交** | AI 能识别各类型输入 → 正确填写 → 提交 → 验证成功提示 | 截图 + 日志 + 断言 |
| **搜索** | AI 能输入关键词 → 点击搜索 → 识别搜索结果 | 截图 + 日志 + 断言 |

### 成功指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 场景通过率 | ≥ 80% | 3 个场景至少 2 个完全通过 |
| 单步推理耗时 | ≤ 10s | AI 单次决策时间 |
| 自愈成功率 | ≥ 50% | 失败后能通过重试恢复 |
| 截图完整率 | 100% | 每步都有截图记录 |
