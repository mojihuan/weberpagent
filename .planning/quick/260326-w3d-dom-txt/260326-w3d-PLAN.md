<objective>
将所有终端日志结构化保存到每个 run 对应的文件夹，DOM txt 也按 run 组织到对应文件夹。

当前问题:
- DOM 文件全部平铺在 outputs/ 目录: `outputs/dom_{run_id}_step{N}.txt`，所有 run 混在一起
- 终端日志只输出到 stderr (Python logging)，没有按 run 保存
- 截图全部平铺在 data/screenshots/

目标结构:
```
outputs/
  {run_id}/
    logs/              # 结构化日志 (JSONL)
      run.jsonl        # 整个 run 的日志
    dom/               # DOM 快照
      step_1.txt
      step_2.txt
    screenshots/       # 截图
      step_1.png
      step_2.png
```

Purpose: 便于调试和回溯每次执行的全过程
Output: RunLogger 工具类 + AgentService 改造
</objective>

<task type="auto">
  <name>Task 1: Create RunLogger utility for per-run structured logging</name>
  <files>backend/utils/run_logger.py</files>
  <action>
Create `backend/utils/run_logger.py` with a `RunLogger` class that:

1. On `__init__(self, run_id: str, base_dir: str = "outputs")`:
   - Creates directory structure: `{base_dir}/{run_id}/logs/`, `{base_dir}/{run_id}/dom/`, `{base_dir}/{run_id}/screenshots/`
   - Opens a JSONL file at `{base_dir}/{run_id}/logs/run.jsonl`
   - Stores `run_id`, base paths as attributes

2. Provides methods:
   - `log(level: str, category: str, message: str, **extra: Any)` - writes one JSONL line with fields: timestamp (ISO), level, category, message, run_id, and any extra fields
   - `log_browser(url: str, dom_content: str, step: int, element_count: int)` - convenience that logs browser state AND saves DOM to `{base_dir}/{run_id}/dom/step_{step}.txt`
   - `log_agent(action_name: str, action_params: dict, reasoning: str, step: int)` - convenience for agent actions
   - `close()` - closes the file handle

3. Use context manager support (`__enter__`/`__exit__`) for safe cleanup.

4. Each `log_browser` call:
   - Writes a JSONL entry with category="browser", fields: url, dom_length, step, element_count, dom_file (relative path)
   - Saves DOM content to `{base_dir}/{run_id}/dom/step_{step}.txt`

5. Each `log_agent` call:
   - Writes a JSONL entry with category="agent", fields: action_name, action_params, reasoning, step

Key patterns from existing code:
- `agent_service.py` line 162-168: current DOM saving logic (`dom_file = self.output_dir / f"dom_{run_id}_step{step}.txt"`)
- `agent_service.py` line 153-195: current logger.info/debug/warning calls with `[{run_id}]` prefix

The RunLogger replaces scattered logger.info calls with structured JSONL entries, making post-run analysis possible.
