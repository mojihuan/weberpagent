<background>
## 问题根因

browser-use 的 DOM 序列化管道有两层过滤机制会将子元素排除：

### 机制 A: Paint Order Removal (最可能的主因)
文件: browser_use/dom/serializer/paint_order.py

`PaintOrderRemover.calculate_paint_order()` 按绘制顺序降序处理节点。
`<tr>` 矩形覆盖子元素矩形 → 子元素被标记 `ignored_by_paint_order = True`。

### 机制 B: Bounding Box Filtering
文件: browser_use/dom/serializer/serializer.py

`_apply_bounding_box_filtering()` → `_should_exclude_child()` 检查子元素是否 99% 在父元素边界内。
`PROPAGATING_ELEMENTS` 列表包含 `a`, `button`, `role="button"` 等，但不包含 `<tr>`。
如果 `<tr>` 上方有传播元素，其边界会向下传播到子元素导致排除。

### 索引分配
文件: serializer.py L617-727

`_assign_interactive_indices_and_mark_new_nodes()` 跳过 `excluded_by_parent` 或 `ignored_by_paint_order` 的节点，
不给它们分配 `backend_node_id` 索引。

## ERP 实际 DOM 结构
- 超链接: `<span class="hand">BU20260330000001 </span>`
- Checkbox: `<span class="el-checkbox__inner"></span>` (Element UI)
- 操作按钮: `<button>收货</button>` (标准 button，已能正常工作)

## Monkey-patch 策略
1. Patch `PaintOrderRemover.calculate_paint_order`: 执行后对含 `hand`/`el-checkbox` class 的节点取消 `ignored_by_paint_order`
2. Patch `DOMTreeSerializer._should_exclude_child`: 含这些 class 的节点永远不排除
3. 在 `browser_agent.py` 和 `proxy_agent.py` 中 Agent 创建前调用 `apply_dom_patch()`
4. 同时保留 prompt 第 7 段中的 evaluate JS 指导作为 fallback
</background>
