## 环境准备

在开始验证前，请确保已配置以下环境变量：

- [ ] 已设置 `ERP_BASE_URL` 环境变量，并指向真实 ERP 服务器地址
- [ ] 已设置 `ERP_USERNAME` 环境变量，并填写有效的 ERP 凭证
- [ ] 已设置 `ERP_PASSWORD` 环境变量，并填写有效的 ERP 凭证
- [ ] 已设置 `WEBSERP_PATH` 环境变量（如果 ERP 系统需要）
- [ ] 已配置 `DASHSCOPE_API_KEY`，用于 AI 执行
- [ ] 后端服务已运行在 8080 端口（`uv run uvicorn backend.api.main:app --reload --port 8080`）
- [ ] 前端服务已运行在 5173 端口（`cd frontend && npm run dev`）
- [ ] 数据库可访问，并且包含测试数据

## AssertionSelector UI 验证（MANUAL-01）

本部分用于验证断言选择器组件 AssertionSelector 是否工作正常。

### Tab 切换验证

- [ ] 任务表单中可见 "API Assertions" 和 "Business Assertions" 两个 Tab
- [ ] 点击 Tab 可以正确切换
- [ ] "API Assertions" Tab 显示代码编辑器
- [ ] "Business Assertions" Tab 显示 "Add Assertion" 按钮

### 模态框基础交互

- [ ] 点击 "Add Assertion" 按钮后打开 AssertionSelector 模态框
- [ ] 模态框标题显示为 "Select Assertion Methods"
- [ ] 模态框有搜索输入框
- [ ] 模态框底部有 "Cancel" 和 "Confirm" 按钮
- [ ] 点击右上角 X 按钮可关闭模态框
- [ ] 按 ESC 键可关闭模态框
- [ ] 点击背景遮罩可关闭模态框

### 断言类分组展开/折叠

- [ ] 断言方法按类分组显示（PcAssert、MgAssert、McAssert 等）
- [ ] 每个分组显示类名和方法数量
- [ ] 点击分组可展开显示方法列表
- [ ] 再次点击分组可折叠方法列表
- [ ] 展开时 ChevronDown 图标旋转 180 度
- [ ] 默认展开第一个类分组

### 搜索功能

- [ ] 搜索框可按方法名过滤
- [ ] 搜索框可按描述内容过滤
- [ ] 清空搜索后恢复完整列表
- [ ] 无匹配结果时显示 "No matching assertion methods found"

### 方法选择

- [ ] 方法名以蓝色等宽字体显示
- [ ] 方法描述以灰色普通字体显示
- [ ] 复选框可切换方法选中状态
- [ ] 可同时选择多个方法
- [ ] 选中后方法名显示在底部 "Selected" 区域
- [ ] 底部显示已选方法数量
- [ ] 未选择方法时 "Confirm" 按钮禁用
- [ ] 选择方法后 "Confirm" 按钮变为可用

### 参数配置 - Headers 下拉

- [ ] Headers 下拉显示选项（main、idle、vice、special、platform、super、camera）
- [ ] 默认选择第一个选项（main）
- [ ] 可切换选择其他 Headers 选项

### 参数配置 - Data 下拉

- [ ] Data 下拉显示方法支持的 data_options
- [ ] 默认选择第一个选项
- [ ] 可切换选择其他 Data 选项

### 参数配置 - 过滤参数（i/j/k）

- [ ] 如果方法有 i/j/k 参数，显示 "Filter Parameters" 区域
- [ ] 参数标签显示参数名
- [ ] 如果参数有选项，显示为下拉框
- [ ] 如果参数无选项，显示为数字输入框
- [ ] 参数描述以问号图标提示

### 已选方法标签管理

- [ ] 已选方法以蓝色标签形式显示
- [ ] 标签格式为 "ClassName.methodName"
- [ ] 点击标签上的 X 可移除该方法
- [ ] 移除后配置区域同步更新

### 确认与取消

- [ ] 点击 "Cancel" 按钮关闭模态框，不保存更改
- [ ] 点击 "Confirm" 按钮关闭模态框，添加断言卡片
- [ ] Confirm 按钮显示已选数量，如 "Confirm (2)"

- [ ] 进入报告列表页面
- [ ] 找到已执行测试对应的报告
- [ ] 点击打开报告详情

### 接口断言结果区域

### 失败状态卡片

- [ ] 失败的断言显示红色背景卡片（bg-red-50）
- [ ] 卡片有红色边框（border-red-200）

- [ ] 显示断言 ID（assertion_id）

- [ ] 显示实际值（actual_value）

### 多断言结果排列

- [ ] 多个断言结果以垂直列表排列
- [ ] 每个断言卡片间距适当
- [ ] 卡片内容完整显示，不被截断

### 无断言结果时的处理

- [ ] 任务无断言配置时，不显示 "接口断言结果" 区域
- [ ] 或显示为空状态提示

## 非 Fail-Fast 验证（MANUAL-04）

本部分用于验证多个断言在执行时不会被第一个失败终止。

### 创建多断言测试任务

**步骤：**
1. 创建新任务
2. 切换到 "Business Assertions" Tab
3. 添加两个断言方法
4. 配置第一个断言使用会失败的参数
5. 配置第二个断言使用会成功的参数
6. 保存任务

### 执行并验证

- [ ] 点击 "Run" 执行任务
- [ ] 等待任务执行完成
- [ ] 打开报告详情
- [ ] 报告中显示两个断言结果（不只是第一个）
- [ ] 第一个断言显示失败状态
- [ ] 第二个断言显示执行结果（成功或失败）
- [ ] 通过率显示为 50%（一成一败）或对应比例

### 关键验证点

- [ ] 第二个断言确实被执行（查看日志或执行步骤）
- [ ] 不是在第一个断言失败后就停止执行
- [ ] 两个断言结果都正确显示在报告中

## 测试场景

### 场景 1：单个断言成功流程

**目的：** 验证单个断言从配置到执行成功的完整流程

**步骤：**
1. 创建新任务，名称为 "Single Assertion Success Test"
2. 配置任务描述（如销售出库流程）
3. 切换到 "Business Assertions" Tab
4. 点击 "Add Assertion" 添加断言
5. 选择一个断言方法，配置合理参数
6. 确认并保存任务
7. 执行任务
8. 等待执行完成
9. 查看报告

**验证：**
- [ ] 断言配置正确保存
- [ ] 任务执行过程中断言被执行
- [ ] 报告显示 "接口断言结果" 区域
- [ ] 断言结果为绿色通过状态
- [ ] 通过率显示 100%

### 场景 2：单个断言失败展示

**目的：** 验证断言失败时的报告展示

**步骤：**
1. 创建新任务
2. 配置一个会失败的断言（如使用错误的参数）
3. 执行任务
4. 查看报告

**验证：**
- [ ] 报告显示红色失败状态卡片
- [ ] 显示错误消息
- [ ] 显示实际值
- [ ] 通过率显示 0%

### 场景 3：多断言非 Fail-Fast

**目的：** 验证多个断言都执行，不因第一个失败而终止

**步骤：**
1. 创建新任务
2. 添加两个断言：
   - 断言 A：配置会失败的参数
   - 断言 B：配置会成功的参数
3. 执行任务
4. 查看报告

**验证：**
- [ ] 报告显示两个断言结果
- [ ] 断言 A 显示失败
- [ ] 断言 B 显示成功（证明第二个断言被执行了）
- [ ] 通过率显示 50%

### 场景 4：无断言配置

**目的：** 验证没有断言配置时的报告展示

**步骤：**
1. 创建不带断言的任务
2. 执行任务
3. 查看报告

**验证：**
- [ ] 报告不显示 "接口断言结果" 区域
- [ ] 报告其他部分正常显示
