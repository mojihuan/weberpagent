| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #ffffff (white) | Modal background, main surfaces |
| Secondary (30%) | #f9fafb (gray-50) | Card backgrounds, section headers |
| Accent (10%) | #3b82f6 (blue-500) | Primary CTA, selected states, links |

Accent reserved for:
- Primary confirm button (`bg-blue-500`)
- Selected checkbox/radio (`text-blue-500`)

- Focus rings (`focus:ring-blue-500`)
- "now" quick button for time fields (`bg-blue-100 text-blue-700`)

## Interaction Patterns

### Search Filtering

| Property | Value |
|----------|-------|
| Debounce | None (immediate with useMemo) |

### Accordion Groups

| Property | Value |
|----------|-------|
| Default expanded | First group only |
| Toggle icon | ChevronDown (rotates 180deg when expanded) |
| Header style | `bg-gray-50 hover:bg-gray-100` |

### Field Selection

| Property | Value |
|----------|-------|
| Selection control | Checkbox |
| Selected state | Checkbox checked + value input visible |
| Value input | Text input for all fields |
| Time field extra | "now" button fills "now" string |

### "now" Button Behavior

| Property | Value |
|----------|-------|
| Appearance | Only for `is_time_field: true` |
| Style | `px-2 py-1 text-xs bg-blue-100 text-blue-700 rounded hover:bg-blue-200` |
| Action | Sets input value to string "now" |
| Semantic | Frontend sends "now", backend converts to current timestamp |

## State Management

### FieldParamsEditor State

```typescript
interface FieldParamsEditorState {
  groups: AssertionFieldGroup[]
  loading: boolean
  error: string | null
  searchQuery: string
  expandedPanels: Set<string>
  selectedFields: Map<string, { name: string; value: string }>
}
```

### Immutable Update Patterns

```typescript
// Toggle panel
setExpandedPanels(prev => {
  const next = new Set(prev)
  if (next.has(name)) next.delete(name)
  else next.add(name)
  return next
})

// Update field value
setSelectedFields(prev => {
  const next = new Map(prev)
  const existing = next.get(fieldName)
  if (existing) {
    next.set(fieldName, { ...existing, value })
  }
  return next
})
```
