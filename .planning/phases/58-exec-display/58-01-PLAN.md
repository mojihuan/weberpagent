    4. In the `'precondition'` event handler, handle two scenarios (per RESEARCH Pitfall 2 + 3):
       - If `prev` is null (precondition events arrive before 'started'), initialize the Run object inline with the precondition data in both `preconditions` and `timeline`.
       - If `prev` exists, check if a precondition with the same `data.index` already exists in `prev.timeline`. If found, REPLACE it (running -> success/failed transition). If not found, APPEND it. This prevents duplicate entries.
       ```typescript
       eventSource.addEventListener('precondition', (e: MessageEvent) => {
         const data: SSEPreconditionEvent = JSON.parse(e.data)
         setRun(prev => {
           if (!prev) {
             // Early precondition — initialize run inline
             return {
               id: runId,
               task_id: '',
               status: 'running',
               started_at: new Date().toISOString(),
               steps: [],
               preconditions: [data],
               api_assertions: [],
               timeline: [{ type: 'precondition', data }],
             }
           }
           const existingIdx = prev.timeline.findIndex(
             item => item.type === 'precondition' && item.data.index === data.index
           )
           const newTimeline = existingIdx >= 0
             ? prev.timeline.map((item, i) => i === existingIdx ? { type: 'precondition' as const, data } : item)
             : [...prev.timeline, { type: 'precondition' as const, data }]
           const newPreconditions = existingIdx >= 0
             ? prev.preconditions?.map((p, i) => p.index === data.index ? data : p) ?? [data]
             : [...(prev.preconditions || []), data]
           return { ...prev, preconditions: newPreconditions, timeline: newTimeline }
         })
       })
       ```

    5. Apply the SAME replace-not-append pattern to the `'api_assertion'` event handler. Map to `{ type: 'assertion', data }` in the timeline.

    Use immutable patterns throughout (spread operators, never mutate). All state updates return new objects.
