    1. Add two imports at the top (after existing imports):

       ```python
       from backend.utils.excel_template import generate_template
       ```

    2. Add the endpoint function AFTER the `get_task_repo` dependency function and BEFORE the `list_tasks` endpoint (around line 18):
       ```python
       @router.get("/template")
       async def download_template():
           """下载测试用例 Excel 模版"""
           buffer = generate_template()
           return StreamingResponse(
               buffer,
               media_type="application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
               headers={"Content-Disposition": "attachment; filename=task_template.xlsx"},
           )
       ```

    CRITICAL: Place the `/template` endpoint BEFORE the `/{task_id}` endpoint. FastAPI evaluates routes in order; if `/{task_id}` comes first, a GET to `/template` would be captured as `task_id="template"`. The `/template` route must be registered first to match correctly.

    The function does NOT need database access (no `Depends(get_task_repo)`). It generates the template fresh on each request using in-memory BytesIO -- no disk I/O.

    Do NOT add the test for this endpoint in this task (API tests require a running server or TestClient setup). The unit tests from Task 1 cover the template content. The endpoint is a thin wrapper (3 lines of logic) that can be manually verified via the /docs page.
