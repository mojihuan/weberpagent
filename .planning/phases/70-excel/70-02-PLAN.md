    **Step 2: Implement type coercion helpers (module-level functions).**

    Per D-09 (lenient mode):

    ```python
    def _coerce_string(value: Any) -> str | None:
        """Coerce cell value to string. Returns None for empty cells."""
        if value is None:
            return None
        if isinstance(value, bool):
            return str(value).lower()  # True -> "true", not "True"
        return str(value).strip()

    def _coerce_int(value: Any, default: int | None = None) -> int | None:
        """Coerce cell value to int. Returns default for empty cells."""
        if value is None:
            return default
        if isinstance(value, bool):
            return None  # boolean is not a valid int input
        if isinstance(value, (int, float)):
            return int(value)
        if isinstance(value, str):
            stripped = value.strip()
            if not stripped:
                return default
            try:
                return int(float(stripped))  # handles "10.0" -> 10
            except (ValueError, TypeError):
                return None
        return None

    def _coerce_json_list(value: Any) -> tuple[list | None, str | None]:
        """Parse JSON array from cell. Returns (parsed_list_or_None, error_or_None)."""
        if value is None:
            return None, None
        raw = str(value).strip()
        if not raw:
            return None, None
        try:
            parsed = json.loads(raw)
            if isinstance(parsed, list):
                return parsed, None
            return None, f"JSON 值不是数组: {raw[:50]}"
        except json.JSONDecodeError:
            return None, f"JSON 格式错误: {raw[:50]}"
    ```

    **Step 3: Implement `parse_excel(buffer: BytesIO) -> ParseResult`.**

    Core logic:
    1. Open workbook with `load_workbook(buffer, data_only=True)` -- `data_only=True` reads formula results, not formulas
    2. Get active sheet (first sheet = "测试用例")
    3. Validate row 1 headers match TEMPLATE_COLUMNS headers. If mismatch, return ParseResult with a single row at row_number=0 containing header mismatch errors
    4. Iterate rows starting from row 2 (`min_row=2`)
    5. For each row:
       a. Check for MergedCell instances. If found, add "合并单元格" error for that row. Still attempt to read other cells.
       b. Check if all cells are None or empty string. If yes, skip this row entirely (do not add to results)
       c. Apply per-column coercion using TEMPLATE_COLUMNS:
          - For columns with key "name", "description", "target_url": use `_coerce_string()`
          - For column with key "max_steps": use `_coerce_int(value, default=10)`
          - For columns with key "preconditions", "assertions": use `_coerce_json_list()`
       d. Check required fields (TEMPLATE_COLUMNS where required=True): if value is None or empty string after coercion, add error "必填字段 '{header}' 不能为空"
       e. If max_steps coercion returned None but cell was not empty, add error "最大步数必须为 1-100 之间的整数"
       f. Collect JSON coercion errors from preconditions/assertions
       g. Build ParsedRow with row_number, data dict, and collected errors
    6. Return ParseResult with all parsed rows, total count, and has_errors flag

    IMPORTANT: Per D-08, the parser must NOT raise exceptions on individual row errors. It collects all errors and returns them. Only truly fatal errors (e.g., completely unopenable file) should raise.

    The data dict values should be:
    - name: str (coerced)
    - description: str (coerced)
    - target_url: str (coerced, default "")
    - max_steps: int (coerced, default 10)
    - preconditions: list | None (parsed JSON, or None if empty/invalid)
    - assertions: list | None (parsed JSON, or None if empty/invalid)

    For rows with JSON parse errors, store the raw string in data[field] so Phase 71 can show what the user typed, but still report the error.
