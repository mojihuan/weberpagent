    **场景 1: AST-01 — 断言 headers 参数传递验证**
    - 标题: `### test_assertion_headers_param - [断言]-headers 参数传递验证`
    - 前置条件: 已登录 ERP 系统，WEBSERP_PATH 已配置
    - 测试步骤:
      1. 在平台 UI 创建测试用例，步骤包含登录操作
      2. 断言配置: class_name=PcAssert, method_name=purchase_order_list_assert, headers='main', data='main'
      3. 执行测试用例，观察断言调用结果
    - 预期: 断言调用成功执行（非 ImportError/HeaderResolutionError），说明 headers='main' 参数被正确传递到 execute_assertion_method 并通过 _process_headers 解析为实际 headers dict
    - 验证条件:
      - 成功: 断言执行结果 success=True（无 HeaderResolutionError），返回 passed=True 或 AssertionError（业务断言结果）
      - 失败: error_type='HeaderResolutionError' 或 'ImportError'
    - Agent 动作说明: 登录 ERP，进入采购管理页面，平台自动调用断言接口验证数据

    **场景 2: AST-02 — inventory_list_data 的 i/j 参数组合验证**
    - 标题: `### test_assertion_ij_params - [断言]-inventory_list_data i/j 参数组合验证`
    - 前置条件: 已登录 ERP 系统，WEBSERP_PATH 已配置，库存数据存在
    - 测试步骤:
      1. 在平台 UI 创建测试用例，步骤包含登录操作
      2. 断言配置: class_name=PcAssert, method_name=inventory_list_data, data='main', api_params={"i": "2", "j": "13"} (i=2=库存中, j=13=待销售)
      3. 执行测试用例，观察断言返回数据是否有效
    - 预期: i/j 参数正确传递到 API 调用，返回有效的库存列表数据（非空列表）
    - 验证条件:
      - 成功: 断言调用返回数据且包含库存记录，success=True
      - 失败: error_type='ParameterError' 或返回数据为空
    - Agent 动作说明: 登录 ERP，进入库存管理页面，平台自动调用断言接口验证 i/j 参数组合

    注意事项:
    - i/j 参数值说明（从 external_precondition_bridge.py 的 _parse_param_options 可知）:
      - i（库存状态）: 1=待入库, 2=库存中, 3=已出库
      - j（物品状态）: 3=待分货, 13=待销售
    - headers 参数标识符: main, idle, vice, special, platform, super, camera
    - data 参数选项: main=主数据, a=配件数据, b=分店数据, c=仓库数据, d=其他数据
