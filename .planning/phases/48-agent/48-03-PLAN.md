  <implementation>
TaskProgressTracker is a @dataclass (NOT frozen, needs mutable internal state):
- `_steps: list[str]` (parsed step descriptions)
- `_completed_steps: set[int]` (indices of completed steps)

ProgressResult is a @dataclass(frozen=True):
- `should_warn: bool`
- `level: str` ("", "warning", "urgent")
- `message: str`
- `remaining_steps: int`
- `remaining_tasks: int`

STEP_PATTERNS: list of compiled regex patterns for parsing:
```python
STEP_PATTERNS = [
    re.compile(r'^\s*Step\s+(\d+)\s*[:：]\s*(.+)$', re.MULTILINE | re.IGNORECASE),
    re.compile(r'^\s*第(\d+)步\s*[:：]?\s*(.+)$', re.MULTILINE),
    re.compile(r'^\s*-\s*\[[\sx]\]\s*(.+)$', re.MULTILINE),
    re.compile(r'^\s*(\d+)[\.、)]\s*(.+)$', re.MULTILINE),
]
```

`parse_task(self, task: str) -> None`:
- Apply each STEP_PATTERN to task text
- First pattern that matches >= 1 step wins (priority order)
- Store step descriptions in _steps (without step numbers)

`check_progress(self, current_step: int, max_steps: int) -> ProgressResult`:
- If _steps is empty, return ProgressResult(should_warn=False, level="", message="", remaining_steps=0, remaining_tasks=0)
- remaining_steps = max_steps - current_step
- remaining_tasks = len(_steps) - len(_completed_steps)
- If remaining_steps <= remaining_tasks: level = "urgent"
- Elif remaining_steps < remaining_tasks * 1.5: level = "warning"
- Else: no warning
- Message format (per D-05, D-06):
  - Warning: `【进度预警】剩余{remaining_steps}步，还有{remaining_tasks}个任务未完成。建议加快节奏，优先完成关键步骤。`
  - Urgent: `【进度紧迫】剩余{remaining_steps}步，还有{remaining_tasks}个任务未完成！建议立即跳过非关键步骤，专注于核心任务。`

`update_from_evaluation(self, evaluation: str) -> None`:
- For each uncompleted step in _steps, take first 3 words of step description
- If ANY of those words appear in evaluation text (case-insensitive), mark step as completed
- This is intentionally loose matching (per RESEARCH.md Open Question 2)
  </implementation>
</feature>
