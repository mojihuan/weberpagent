```python
@dataclass
class StallDetector:

```

5. `check(self, action_name: str, target_index: int | None, evaluation: str, dom_hash: str) -> StallResult`:
   - Create _StepRecord, append to _history
   - Return self._detect_stall()

6. `_detect_stall(self) -> StallResult`:
   - Check _check_consecutive_failures() first (higher priority)
   - If not intervening, check _check_stagnant_dom()

7. `_check_consecutive_failures(self) -> StallResult`:
   CRITICAL: Avoid Pitfall 6 from RESEARCH. Initialize from first record:
   ```python
   consecutive = 0
   last_action = None
   last_index = None
   for record in reversed(self._history):
       is_failure = bool(FAILURE_KEYWORDS.search(record.evaluation))
       if not is_failure:
           break  # Success resets
       if last_action is None:
           last_action = record.action_name
           last_index = record.target_index
           consecutive = 1
           continue
       if record.action_name != last_action or record.target_index != last_index:
           break
       consecutive += 1
   ```
   If consecutive >= max_consecutive_failures, return StallResult with Chinese message:
   `【停滞警告】连续{consecutive}次对元素#{last_index}执行'{last_action}'失败。建议：使用evaluate执行JavaScript直接操作DOM、使用find_elements查找真实input元素、或滚动页面查找其他元素。`

8. `_check_stagnant_dom(self) -> StallResult`:
   - Take last max_stagnant_steps records from _history
   - If fewer than max_stagnant_steps records, return no intervention
   - Extract dom_hash from each into a set
   - If set size == 1 (all identical), return StallResult with Chinese message:
   `【页面停滞】连续{max_stagnant_steps}步页面无变化（DOM指纹相同）。建议：滚动页面、点击展开折叠区域、或检查是否需要切换到其他iframe。`
