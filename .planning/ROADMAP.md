## Phase Details

### Phase 120: 删除测试基础设施
**Goal**: 代码库中不存在任何测试代码、测试配置和测试依赖，应用启动和运行不受影响
**Depends on**: Nothing (first phase of milestone)
**Requirements**: TEST-01, TEST-02, TEST-03, TEST-04
**Success Criteria** (what must be TRUE):
  1. tests/ 目录完全删除，项目中不存在任何测试文件或 conftest.py
  2. pyproject.toml 和 requirements 文件中不含 pytest、pytest-asyncio、pytest-timeout、httpx 等测试依赖
  3. 无 pytest.ini 或 pyproject.toml [tool.pytest] 配置段存在
  4. 源代码中无仅用于测试的引用（# pytest 标记、test-only helper、mock fixture import）
  5. FastAPI 应用正常启动，无 ImportError
**Plans**: 2 plans

Plans:
- [x] 120-01: 删除 tests/ 目录和 pytest 配置
- [x] 120-02: 清理测试依赖和源码测试引用

### Phase 121: 死代码清理
**Goal**: 代码库中不存在任何未使用的 import、函数、变量或废弃模块
**Depends on**: Phase 120
**Requirements**: DEAD-01, DEAD-02, DEAD-03, DEAD-04
**Success Criteria** (what must be TRUE):
  1. 所有 Python 文件中无未使用的 import 语句（可用 pyflakes 验证零警告）
  2. 不存在未被任何调用方引用的函数或方法
  3. 不存在未引用的模块级变量或常量
  4. 不存在整个文件未被 import 的废弃模块文件
  5. FastAPI 应用正常启动，无 ImportError
**Plans**: 2 plans

Plans:
- [x] 121-01: 扫描并删除未使用 import 和废弃模块文件
- [x] 121-02: 清理未调用函数和未引用变量

### Phase 122: 重复逻辑合并
**Goal**: 代码库中不存在相同或高度相似的函数、错误处理模式和序列化逻辑
**Depends on**: Phase 121
**Requirements**: DUP-01, DUP-02, DUP-03
**Success Criteria** (what must be TRUE):
  1. 不存在相同或高度相似（>80% 代码重复）的函数，已合并为共享函数
  2. 错误处理模式统一（相同场景使用相同的 try-except 结构和异常类型）
  3. 数据转换和序列化逻辑统一（无散落在多处的相同转换代码）
  4. FastAPI 应用正常启动，无 ImportError
**Plans**: 2 plans

Plans:
- [ ] 122-01: 识别并合并重复函数和序列化逻辑
- [ ] 122-02: 统一错误处理模式

### Phase 123: 命名规范化与类型标注
**Goal**: 所有公共 API 使用一致的 snake_case 命名和完整的类型标注
**Depends on**: Phase 122
**Requirements**: NAME-01, NAME-02, NAME-03, TYPE-01, TYPE-02, TYPE-03
**Success Criteria** (what must be TRUE):
  1. 所有公共函数使用一致的 snake_case 命名，无命名风格偏差
  2. 变量命名语义清晰，无歧义缩写
  3. 文件和模块命名符合 Python 包命名规范
  4. 所有公共函数的参数和返回值具有完整类型标注
  5. 核心数据模型字段具有完整类型标注，py.typed 标记文件存在
  6. FastAPI 应用正常启动，无 ImportError
**Plans**: 2 plans

Plans:
- [ ] 123-01: 统一命名风格（函数、变量、模块）
- [ ] 123-02: 补全类型标注并添加 py.typed

### Phase 124: 函数/模块优化
**Goal**: 所有函数职责单一、长度合理、嵌套不超 4 层，模块高内聚低耦合
**Depends on**: Phase 123
**Requirements**: FUNC-01, FUNC-02, FUNC-03, FUNC-04
**Success Criteria** (what must be TRUE):
  1. 不存在超过 50 行的函数（已拆分为单一职责小函数）
  2. 模块职责清晰，高内聚低耦合（无跨层调用或职责混乱的模块）
  3. 条件逻辑嵌套不超过 4 层，复杂条件已简化
  4. 错误处理模式统一，无重复 try-except 块
  5. FastAPI 应用正常启动，无 ImportError
**Plans**: 2 plans

Plans:
- [ ] 124-01: 拆分过长函数和简化嵌套逻辑
- [ ] 124-02: 重组模块职责和统一异常处理
