## Phase Details

### Phase 116: 自愈模块删除
**Goal**: 四个自愈模块从代码库中彻底消失，所有导入引用清理干净，项目可正常启动
**Depends on**: Nothing (first phase of milestone)
**Requirements**: REMOVE-01, REMOVE-02, REMOVE-03, REMOVE-04
**Success Criteria** (what must be TRUE):
  1. self_healing_runner.py / llm_healer.py / error_classifier.py / healer_error.py 四个文件从 backend/core/ 中删除
  2. 项目中无任何文件包含对这四个模块的 import 语句
  3. FastAPI 应用可以正常启动（uvicorn 无 ImportError）
  4. grep -r "self_healing_runner\|llm_healer\|error_classifier\|healer_error" backend/ 返回零匹配（排除测试文件）
**Plans**: 2 plans

Plans:
- [x] 116-01: 删除四个自愈模块文件
- [x] 116-02: 清理所有导入引用和残余代码

### Phase 117: 管道与数据层简化
**Goal**: 代码执行管道变为一次性 pytest 运行，数据库和 schema 不再包含 healing 字段
**Depends on**: Phase 116
**Requirements**: SIMPLIFY-01, SIMPLIFY-02, SIMPLIFY-03, DB-01, DB-02, DB-03
**Success Criteria** (what must be TRUE):
  1. POST /execute-code 执行 pytest 一次即返回结果（passed/failed），无重试循环
  2. runs.py 中不再引用 SelfHealingRunner、LLMHealer、ErrorClassifier
  3. StepCodeBuffer.append_step_async() 方法已移除，同步 append_step() 保留
  4. Run 数据库模型不再包含 healing_status/healing_attempts/healing_error/healing_error_category 字段
  5. RunResponse 和 ReportDetailResponse schema 无 healing 相关字段
**Plans**: 2 plans

Plans:
- [x] 117-01: 简化 runs.py 执行管道 + StepCodeBuffer/code_generator 清理
- [x] 117-02: DB 模型和 schema healing 字段清理

### Phase 118: API 与前端清理
**Goal**: API 端点反映简化后的执行模型，前端不再展示任何 healing 相关 UI
**Depends on**: Phase 117
**Requirements**: API-01, API-02, FE-01, FE-02, FE-03, FE-04, FE-05
**Success Criteria** (what must be TRUE):
  1. POST /execute-code 返回 passed/failed 状态，无 healing 状态轮询
  2. GET /runs/{run_id}/code 正常返回代码内容，不受影响
  3. 前端 StatusBadge 不再显示 healing_* 状态标签
  4. CodeViewerModal 不再轮询 healing 状态，只展示执行结果
  5. 前端 TypeScript 类型定义中无 healing 相关字段
**Plans**: 2 plans
**UI hint**: yes

Plans:
- [x] 118-01: API 端点简化
- [x] 118-02: 前端 healing UI 和类型清理
