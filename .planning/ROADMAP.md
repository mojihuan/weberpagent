## Phase Details

### Phase 130: 安全与关键正确性修复
**Goal**: 系统不再有路径遍历安全漏洞，停滞检测和进度追踪工作正确
**Depends on**: Nothing (first phase, highest priority fixes)
**Requirements**: CORR-02, CORR-01, CORR-03
**Success Criteria** (what must be TRUE):
  1. execute_run_code 拒绝包含路径遍历序列（如 ../）的文件路径，返回明确的错误响应
  2. StallDetector 阈值不再因双重调用而被意外减半 — 单次运行中停滞检测行为与配置一致
  3. TaskProgressTracker 不再在两处被重复调用 — 进度追踪结果与实际步骤数一致
**Plans**: 2 plans

Plans:
- [x] 130-01-PLAN.md — TDD: 路径遍历防护 + 测试基础设施 (CORR-02)
- [x] 130-02-PLAN.md — TDD: 删除 create_step_callback 死代码 + 验证测试 (CORR-01, CORR-03)

### Phase 131: 后端内存与错误处理
**Goal**: 后端 SSE 管道不再因异常中断，EventManager 不再无限增长内存，断言检测返回真实结果
**Depends on**: Phase 130 (correctness fixes establish correct baseline)
**Requirements**: MEM-01, MEM-02, MEM-04, ERR-01, ERR-03
**Success Criteria** (what must be TRUE):
  1. 运行完成后 EventManager._events dict 被自动清理，多次运行不会导致内存持续增长
  2. SSE event_generator 中 publish 失败不再中断整个 pipeline — 后续事件正常推送
  3. assertion_service.check_element_exists 返回基于真实 DOM 检测的结果（不再硬编码 True）
  4. EventManager heartbeat task 在重新订阅时旧 task 被正确取消，无 task 泄漏
**Plans**: 2 plans

Plans:
- [x] 131-01-PLAN.md — EventManager 内存泄漏修复 + StallDetector 历史上限 (MEM-01, MEM-02, MEM-04)
- [ ] 131-02-PLAN.md — SSE 异常保护 + 断言 DOM 检测实现 (ERR-01, ERR-03)

### Phase 132: 后端异步与状态
**Goal**: 后端不再有阻塞事件循环的操作，外部断言 context 不再泄漏到 variable_map
**Depends on**: Phase 131 (memory fixes establish clean event loop baseline)
**Requirements**: ASYNC-01, ASYNC-02, STATE-01
**Success Criteria** (what must be TRUE):
  1. 截图保存操作不阻塞事件循环 — save_screenshot 执行期间其他 async 操作正常响应
  2. 代码执行子进程使用 async API — execute_code 运行期间后端仍可处理 SSE 心跳和其他请求
  3. external_assertion_summary 不再出现在 variable_map 中 — 前置条件变量替换不受断言结果污染
**Plans**: 2 plans

Plans:
- [ ] 132-01: 阻塞操作异步迁移 (ASYNC-01, ASYNC-02)
- [ ] 132-02: 外部断言状态解耦 (STATE-01)

### Phase 133: 前端健壮性
**Goal**: 前端 SSE 连接不再因解析错误崩溃，长时间运行不因数组拷贝变慢，组件状态更新符合 React immutable 模式
**Depends on**: Phase 131 (backend SSE fixes establish correct event format)
**Requirements**: MEM-03, ERR-02, ERR-04, STATE-02
**Success Criteria** (what must be TRUE):
  1. useRunStream 接收到格式错误的 JSON 数据时不再抛出未捕获异常 — SSE 连接保持正常
  2. 运行超过 100 步时 useRunStream 的 steps/timeline 更新不产生 O(n^2) 性能退化
  3. isConnected 状态仅在 EventSource 连接真正建立后变为 true — UI 不再显示虚假的"已连接"状态
  4. 前端组件 useState 更新使用 immutable 模式 — React 渲染检测正确触发
**UI hint**: yes
**Plans**: 2 plans

Plans:
- [ ] 133-01: useRunStream 健壮性与性能修复 (MEM-03, ERR-02, ERR-04)
- [ ] 133-02: 前端 immutable 更新模式 (STATE-02)

### Phase 134: 死代码清理
**Goal**: 代码库不再包含完全未使用的代码，前端数据获取统一使用 React Query，后端错误处理策略一致
**Depends on**: Phase 132, Phase 133 (all functional fixes complete before cleanup)
**Requirements**: DEAD-01, DEAD-02, DEAD-03, DEAD-04
**Success Criteria** (what must be TRUE):
  1. response.py 中 85 行未使用代码被删除 — ruff/pyflakes 扫描零新增警告
  2. useTasks, useReports, useDashboard, useBatchProgress 四个 hooks 使用 React Query useQuery — 消除手动 fetch/loading/error boilerplate
  3. PreSubmitGuard 中不可达的死代码路径被清理 — 代码审计无不可达分支
  4. 后端可选操作统一使用 non_blocking_execute — 全局 grep 无其他错误处理模式混用
**UI hint**: yes
**Plans**: 2 plans

Plans:
- [ ] 134-01: 后端死代码与错误处理统一 (DEAD-01, DEAD-03, DEAD-04)
- [ ] 134-02: React Query 迁移 (DEAD-02)
