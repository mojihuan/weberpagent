## Phase Details

### Phase 102: 执行修复
**Goal**: AI 生成的 Playwright 测试代码可以通过 pytest 子进程成功启动执行，不会被参数错误、语法错误或开发环境热重载阻断
**Depends on**: Nothing (first phase in milestone, builds on v0.10.5 code generation fixes)
**Requirements**: EXEC-01, EXEC-02, EXEC-03
**Success Criteria** (what must be TRUE):
  1. SelfHealingRunner 调用 pytest 时不传 `--headed` 或任何 headed/headless 相关参数，pytest 默认以 headless 模式运行
  2. AI 执行完成后生成的测试代码文件可通过 `python -c "import ast; ast.parse(open(file).read())"` 无 SyntaxError
  3. 开发环境 `uvicorn --reload` 启动时，conftest.py 写入 outputs 目录不再触发服务器热重载
  4. 现有 967 个单元测试全部通过，无回归
**Plans**: 1 plan

Plans:
- [ ] 102-01-PLAN.md — 修复 pytest 参数/注释换行/热重载干扰 (EXEC-01, EXEC-02, EXEC-03)

### Phase 103: 自愈改进
**Goal**: SelfHealingRunner 在执行失败时智能区分"执行环境错误"和"代码错误"，环境错误直接报告而非浪费 LLM 调用尝试修复
**Depends on**: Phase 102
**Requirements**: HEAL-01
**Success Criteria** (what must be TRUE):
  1. pytest 退出码为 2 (test interrupted)、4 (pytest error) 或 5 (no tests collected) 时，SelfHealingRunner 跳过 LLM 修复并返回分类后的错误信息
  2. pytest 退出码为 1 (test failures) 且 stderr 包含 `E   SyntaxError:` 或 `E   ImportError:` 时，归类为代码错误并尝试 LLM 修复
  3. pytest 退出码为 1 但 stderr 不匹配任何已知模式时，默认归类为 CODE_RUNTIME（可修复类别），不遗漏 LLM 修复机会
  4. 错误分类器是纯函数，有独立单元测试覆盖全部 8 种分类路径
**Plans**: TBD

### Phase 104: E2E 验证
**Goal**: 确认整个测试代码执行管线端到端工作正常 — AI 执行任务、生成代码、pytest 运行、结果返回全流程通畅
**Depends on**: Phase 103
**Requirements**: E2E-01
**Success Criteria** (what must be TRUE):
  1. AI 执行一个完整的测试任务后，生成的 Playwright 测试代码可被 pytest 子进程成功执行
  2. pytest 执行返回退出码 0 (pass) 或 1 (test-level fail)，而非退出码 2/4/5 (环境/配置错误)
  3. 执行结果（pass 或 fail 原因）可通过 API 查询到，报告信息有意义
