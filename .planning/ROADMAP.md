## v0.10.5 生成测试代码修复与优化 (In Progress)

**Milestone Goal:** 修复代码生成系统，使 AI 执行后自动生成的 Playwright 测试代码可直接运行

- [x] **Phase 99: 核心键名修复** - 修复 action_translator 的键名不匹配，使 click/input 正确生成 Playwright 代码 (completed 2026-04-24)
- [ ] **Phase 100: 操作翻译扩展** - 补充 wait/select_dropdown/evaluate/upload_file 等 12 种缺失操作翻译
- [ ] **Phase 101: 测试验证** - 单元测试 + 端到端验证代码生成质量

## Phase Details

### Phase 99: 核心键名修复
**Goal**: AI 执行步骤中的 click/input action 能正确翻译为可执行的 Playwright 代码
**Depends on**: Nothing (first phase in milestone)
**Requirements**: KEY-01, KEY-02, KEY-03, KEY-04
**Success Criteria** (what must be TRUE):
  1. ActionTranslator 处理 browser-use 输出的 "click" action 时生成 `page.locator("xpath=...").click()` 代码
  2. ActionTranslator 处理 browser-use 输出的 "input" action 时生成 `page.locator("xpath=...").fill("...")` 代码
  3. _heal_weak_steps() 对 click/input 步骤触发 LLM healing，而非静默跳过
  4. _build_fallback_code 和 _build_llm_only_code 对 click/input 生成有意义代码而非空注释
**Plans**: 2 plans

Plans:
- [x] 99-01-PLAN.md — 重命名 _CORE_TYPES, translate(), translate_with_llm() 及对应测试中的 click_element/input_text 为 click/input
- [x] 99-02-PLAN.md — 修复 _heal_weak_steps() 类型检查及 code_generator/healer 测试中的键名

### Phase 100: 操作翻译扩展
**Goal**: 所有 browser-use 输出的非常规 action 类型都能翻译为有意义的 Playwright 代码或注释
**Depends on**: Phase 99
**Requirements**: TRANS-01, TRANS-02, TRANS-03, TRANS-04, EDGE-01, EDGE-02, EDGE-03, EDGE-04, EDGE-05, EDGE-06, EDGE-07, EDGE-08
**Success Criteria** (what must be TRUE):
  1. wait action 生成 `page.wait_for_timeout(seconds * 1000)` 可执行代码
  2. select_dropdown action 生成 `page.locator("xpath=...").select_option(text)` 使用元素定位器
  3. evaluate action 生成 `page.evaluate(code)` 显示 JavaScript 执行代码
  4. upload_file action 生成 `page.locator("xpath=...").set_input_files(path)` 使用元素定位器
  5. 其余 8 种边缘 action (switch/close/search_page/screenshot/done/search/dropdown_options/extract) 生成有意义的注释而非空行
**Plans**: TBD

Plans:
- [ ] 100-01: 常用操作翻译 (wait/select_dropdown/evaluate/upload_file)
- [ ] 100-02: 边缘操作翻译 (switch/close/search_page 等 8 种)

### Phase 101: 测试验证
**Goal**: 通过单元测试和端到端验证确认代码生成系统对真实 browser-use 输出正确工作
**Depends on**: Phase 100
**Requirements**: TEST-01, TEST-02, TEST-03, TEST-04
**Success Criteria** (what must be TRUE):
  1. 单元测试使用 click/input 键名（而非 click_element/input_text）验证翻译输出
  2. wait/select_dropdown/evaluate 翻译有独立单元测试覆盖
  3. _heal_weak_steps() 单元测试验证 click/input 键名触发 LLM healing 路径
  4. AI 执行完整任务后，生成的代码文件包含可执行的 Playwright click/input 调用
**Plans**: TBD

Plans:
- [ ] 101-01: 单元测试 — 键名修复 + 新翻译覆盖
- [ ] 101-02: 端到端验证 — 完整任务执行后生成可运行代码
