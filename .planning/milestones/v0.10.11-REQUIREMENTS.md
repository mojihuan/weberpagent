# Requirements: aiDriveUITest v0.10.11

**Defined:** 2026-04-29
**Core Value:** 让 QA 用自然语言写测试用例，AI 自动执行并生成报告

## v1 Requirements

### 后端模块删除

- [x] **REMOVE-01**: 删除 `backend/core/self_healing_runner.py` — 移除三层自愈管道（pytest 重试 + LLM 修复循环）
- [x] **REMOVE-02**: 删除 `backend/core/llm_healer.py` — 移除 LLM 修复接口（heal + repair_code）
- [x] **REMOVE-03**: 删除 `backend/core/error_classifier.py` — 移除 pytest 错误分类器
- [x] **REMOVE-04**: 删除 `backend/core/healer_error.py` — 移除自定义异常

### 后端管道简化

- [x] **SIMPLIFY-01**: 简化 `runs.py` — 用简单的一次性 pytest 执行替代 SelfHealingRunner 重试循环，保留 storage_state 登录和 conftest 生成
- [x] **SIMPLIFY-02**: 移除 `StepCodeBuffer.append_step_async()` 内联修复方法，保留同步 append_step()
- [x] **SIMPLIFY-03**: 清理 `code_generator.py` — 移除 HealerError 导入和 healer logger 引用

### 数据层清理

- [x] **DB-01**: 移除 Run 模型 healing_status/healing_attempts/healing_error/healing_error_category 四个字段
- [x] **DB-02**: 移除 RunRepository.update_healing_status() 方法
- [x] **DB-03**: 清理 RunResponse 和 ReportDetailResponse schema 中 healing 相关字段

### API 端点

- [x] **API-01**: 简化 POST /execute-code — 保留端点，改为一次性 pytest 执行（无重试、无 healing 状态轮询），返回 passed/failed
- [x] **API-02**: 保留 GET /runs/{run_id}/code — 代码查看功能不受影响

### 前端清理

- [x] **FE-01**: 移除 StatusBadge 中 healing_* 状态标签
- [x] **FE-02**: 简化 CodeViewerModal — 移除 healing 状态轮询，简化执行/结果展示
- [x] **FE-03**: 清理 types/index.ts 和 API 类型中 healing 字段
- [x] **FE-04**: 清理 ReportDetail.tsx healingStatusMap
- [x] **FE-05**: 移除 CodeExecutionStatus.tsx healing 相关状态（如文件独立存在则删除）

### 测试清理

- [x] **TEST-01**: 删除自愈测试文件（test_self_healing_runner, test_llm_healer, test_healer_error, test_error_classifier, test_e2e_healing_pipeline）
- [x] **TEST-02**: 清理其他测试中 healing 相关 mock 和断言
- [x] **TEST-03**: 全量回归测试通过

## Out of Scope

| Feature | Reason |
|---------|--------|
| 代码生成管道重构 | StepCodeBuffer/ActionTranslator 保持不变，只移除 healing 部分 |
| 替换自愈为其他方案 | 直接删除，不做替代 |
| 前端代码编辑器 | 只读查看保持不变 |
