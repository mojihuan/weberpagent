目标表格：
| 目标 | 描述 |

**OPTIMIZE-04: 失败恢复策略（Per D-12）：**

目标表格：
| 目标 | 描述 |
|------|------|
| 为三种失败模式定义统一的检测→恢复规则 | 点击无 DOM 变化 / 误点错误列 / 编辑态误判 |
| 规则表格式：失败模式 → 检测条件 → 切换动作 | 统一模板便于实施 |

设计规则表（统一格式: 失败模式 / 检测条件 / 切换动作 / 示例）：

规则 1 — 点击无 DOM 变化:
- 失败模式: Agent 点击某 index 后 DOM hash 未变化
- 检测条件: step_callback 中 `dom_hash_before == dom_hash_after` 且 `action_name == "click"`
- 切换动作: (1) 标注该 index 为 `<!-- 点击无响应 -->` (2) 如果是 td，建议先检查是否需要滚动 (3) 如果是 input，切换到 evaluate JS 策略
- 示例: Agent 点击 index=5 的 td 后无变化 → 标注 `<!-- 点击无响应，尝试 evaluate JS: document.querySelector('...').click() -->`

规则 2 — 误点错误列:
- 失败模式: Agent 点击了错误的列（如点了总成本列而非销售金额列）
- 检测条件: evaluation 中包含"wrong column"/"错误列"/"误点"等关键词，或 DOM 变化显示错误的 input 被激活
- 切换动作: (1) 标注误点元素为 `<!-- 非目标列，请忽略 -->` (2) 标注目标列元素为 `<!-- 目标列，请点击此处 -->` (3) 利用 OPTIMIZE-01 的行标识帮助 Agent 重新定位
- 示例: Agent 点了总成本 td → 标注 `<!-- 非目标列 -->` → 销售金额 td 标注 `<!-- 目标列: 销售金额 -->`

规则 3 — 编辑态判断失误:
- 失败模式: Agent 误认为 td 已进入编辑态（input 已出现），实际 input 未出现
- 检测条件: Agent 尝试 input 操作但元素不可交互，或 evaluation 包含"not editable"/"无法输入"
- 切换动作: (1) 标注当前 td 为 `<!-- 编辑态未激活，需先 CLICK -->` (2) 提供 click-then-wait-retry 工作流指导 (3) 如果 click 后仍不出现，降级到 evaluate JS
- 示例: Agent 尝试 INPUT 但失败 → 标注 `<!-- 编辑态未激活，执行: CLICK td → 等待 1s → INPUT -->`

集成点：
- DOM Patch 层: 与 OPTIMIZE-02 的 `_patch_dynamic_annotation()` 共用 — 失败恢复标注是动态标注的子类型
  - 规则 1 的检测：StallDetector 已有 dom_hash 对比能力，新增 click 无效检测条件
  - 规则 2 的检测：需要 evaluation 关键词匹配 + OPTIMIZE-01 行标识的辅助
  - 规则 3 的检测：需要 action_name + evaluation + 元素状态综合判断
