Step 1: Read all P2 and P3 files. Audit each file.

### tasks.py Audit Checklist (P2):
1. **POST /api/tasks** (create_task): Parameter validation on name, description, steps, etc. Are string lengths constrained? Is the steps format validated?
2. **GET /api/tasks** (list_tasks): No pagination? What happens with 1000+ tasks?
3. **GET /api/tasks/{task_id}** (get_task): Is task_id validated (format, length)? Or is it just a bare `str`?
4. **PUT /api/tasks/{task_id}** (update_task): Same task_id validation question. What about partial updates?
5. **DELETE /api/tasks/{task_id}** (delete_task): Is there cascade deletion? What happens to associated runs?
6. **POST /api/tasks/import** (import_tasks): File upload validation -- does it use `_validate_upload_file()`? What file types are accepted? Size limits?
7. **POST /api/tasks/import/confirm** (confirm_import): How is the import session validated?
8. **Error handling**: Are all CRUD operations wrapped in try/except? Do they use raise_not_found consistently?

### reports.py Audit Checklist (P2):
1. **GET /api/reports** (list_reports): Pagination? Filtering?
2. **GET /api/reports/{report_id}** (get_report): Is report_id validated?
3. **GET /api/reports/{report_id}/download** (download_report): FileResponse -- is the path validated? Same path traversal concern as screenshots in runs_routes.py?
4. **Error handling**: Consistent error responses?

### runs.py Audit (P2):
This is just a backward-compat re-export file (7 lines). Quick check: What does it re-export? Is it still needed?

### dashboard.py Audit Checklist (P3):
1. **GET /api/dashboard/stats** (get_stats): Any input parameters? Error handling for empty database?
2. **GET /api/dashboard/recent-runs** (get_recent_runs): Any limits? What if there are thousands of runs?
3. **Error handling**: Does it handle database errors gracefully?

### response.py Audit (P3):
1. Are `success_response()` and `error_response()` actually used by any route? (RESEARCH says NO)
2. Are `ErrorCodes` constants used anywhere?
3. If unused, document as dead code finding.

### __init__.py Audit (P3):
1. Only imports `tasks` and `runs` -- but main.py imports 8 route modules directly. Is this a maintenance gap?
