**Part A: P2 Supporting Services Review**

For each P2 file, read the full source and document findings:

**precondition_service.py (378 lines):**
- exec() security: Verify the execution environment includes `__builtins__` (full Python runtime). Check what the exec sandbox provides and what it does not restrict.
- ContextWrapper state: Verify how context variables are stored and passed to the pipeline. Is there state leakage between runs?
- async executor: Verify the `asyncio.wait_for()` timeout (30s per D). Check for paths where timeout does not apply.
- Jinja2 variable substitution: Verify error handling for undefined variables, circular references, or malformed templates.

**stall_detector.py (221 lines):**
- History accumulation: Verify `_history` list grows without bounds. Confirm typical max_steps (10-30) makes this low risk.
- Stall detection logic: Verify consecutive failure counting and DOM hash comparison.
- Detector result immutability: Verify `StallResult` is frozen dataclass.

**assertion_service.py (157 lines):**
- Element check stub (Pitfall 8): Verify `check_element_exists()` always returns True when `history.is_done`. No actual DOM checking occurs.
- URL assertion: Verify `url_contains` checks the final URL correctly.
- Text assertion: Verify `text_exists` checks page content correctly.
- Error assertion: Verify `no_errors` assertion logic.

**event_manager.py (156 lines):**
- Memory leak: Verify `_events` dict grows indefinitely, `cleanup()` is never called.
- Heartbeat task leak (Pitfall 5): Verify `_heartbeat_tasks` dict. If subscribe() raises before finally, heartbeat task may not be cancelled. Multiple subscribe() calls for same run_id overwrite previous task reference.
- History replay: Verify late subscribers receive all events correctly.

**test_flow_service.py (193 lines):**
- Variable substitution: Verify two-phase substitution (precondition variables -> task description variables).
- Step number shifting: Verify login prefix injection correctly shifts step numbers.
- Edge cases: empty task steps, missing variables, special characters in variable values.

**batch_execution.py (107 lines):**
- Semaphore concurrency: Verify asyncio.Semaphore gates parallel execution correctly.
- Fire-and-forget: Verify `asyncio.create_task()` without tracking (from CONCERNS.md).
- Semaphore internal access (Pitfall 9): Verify `self._semaphore._value` accesses private attribute.
