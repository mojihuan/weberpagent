    ## P2 Quick-Scan Strategy

    For each P2 file, read the content and check for:

    ### Correctness checks (per CORR-03):
    - State management errors: stale closures, missing deps in useEffect/useCallback, state that should be derived
    - Event handler bugs: missing preventDefault, incorrect event type, missing cleanup
    - Data flow issues: props drilling problems, inconsistent type usage, missing null checks
    - API response handling: missing error states, unhandled edge cases, race conditions in async operations

    ### Performance checks (per D-04 PERF-02):
    - Missing React.memo on components that receive stable props
    - Missing useMemo for expensive computations (filtering, sorting, transforming arrays)
    - Missing useCallback for functions passed as props
    - Large lists without virtualization (check list rendering in TaskTable, StepTimeline, Reports)
    - React Query misconfiguration (check if useQuery/useMutation is used anywhere or only manual fetch)
    - Too-coarse state updates (one setState that causes full component re-render)
    - set-state-in-effect anti-pattern (ESLint flagged in RunMonitor.tsx line 26 and CodeViewerModal.tsx line 26)

    ## Specific file-by-file checklist:

    ### types/index.ts (456 lines)
    - Check for `any` type usage (ESLint flagged 4 instances at lines 91, 334, 339, 353)
    - Verify TypeScript interfaces match backend Pydantic schemas (compare Task, Run, Step types with backend/db/schemas.py)
    - Check for inconsistent optional field handling (T | undefined vs T | null)
    - Look for type assertions (`as`) that bypass type safety

    ### hooks/useTasks.ts, useReports.ts, useDashboard.ts (manual fetch pattern)
    - Verify these use useState + useCallback + useEffect pattern (NOT React Query useQuery per RESEARCH)
    - Check useCallback dependency arrays for completeness
    - Check useEffect cleanup (does it abort in-flight fetches on unmount?)
    - Identify the React Query gap: QueryClientProvider configured in App.tsx but no useQuery calls exist
    - Document this as a finding -- CONVENTIONS.md claims React Query but it's unused

    ### hooks/useBatchProgress.ts (68 lines)
    - Check the polling pattern (setInterval 2s per RESEARCH)
    - Verify cleanup (clearInterval on unmount)
    - Check for memory leaks (interval not cleared when component unmounts during fetch)

    ### pages/RunMonitor.tsx (132 lines)
    - ESLint flagged set-state-in-effect at line 26: `setViewIndex(run.timeline.length - 1)` inside useEffect
    - What triggers this effect? Every timeline update?
    - Could this cause cascading renders? (update timeline -> effect fires -> setState -> re-render -> effect fires?)
    - Verify useRunStream usage is correct

    ### components/RunMonitor/StepTimeline.tsx (285 lines)
    - Check if timeline items use `key={index}` per RESEARCH anti-pattern
    - Large list rendering: for 50+ steps, is there virtualization?
    - Missing React.memo on timeline item components?

    ### components/Report/TimelineItemCard.tsx (247 lines)

    - Verify screenshot URL handling

    ### components/TaskDetail/CodeViewerModal.tsx (175 lines)

    - Verify the effect pattern
