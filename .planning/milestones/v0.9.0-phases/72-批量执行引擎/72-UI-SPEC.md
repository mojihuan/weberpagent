## Interaction Contracts

### BatchExecuteDialog

**Trigger:** User clicks "批量执行" button in `BatchActions` bar.

**Layout:**
```
+------------------------------------------+
|  批量执行确认                        [X]  |
|                                          |
|  已选择 5 个任务进行批量执行              |
|                                          |
|  并发数                                  |
|  [====O=====]  2                         |
|   1        4  (range indicator)          |
|                                          |
|  每个任务将独立执行，单个任务失败不影响   |
|  其他任务继续运行。                       |
|                                          |
|               [取消]  [开始执行]          |
+------------------------------------------+
```

**States:**

| State | Visual | Behavior |
|-------|--------|----------|
| Default | Confirm button enabled (blue) | Slider shows current value |
| Confirming | Confirm button shows spinner + disabled | Dialog stays open, buttons disabled |
| Error | Toast notification (sonner) | Dialog stays open, buttons re-enabled |
| Success | Toast success + dialog closes | Auto-close after batch creation confirmed |

**Slider specs:**
- Range: 1 to 4 (integers only)
- Default: 2
- Step: 1
- Width: full dialog content width
- Active track: blue-500
- Thumb: white with blue-500 border
- Current value displayed to the right of slider as a number
- Min/max labels below track: "1" on left, "4" on right

**Props:**
```typescript
interface BatchExecuteDialogProps {
  open: boolean
  taskCount: number
  onConfirm: (concurrency: number) => void
  onCancel: () => void
  loading?: boolean
}
```

### BatchActions Extension

**New button specs:**
- Icon: `Play` from lucide-react (`w-4 h-4`)
- Label: "批量执行"
- Style: `flex items-center gap-1 px-3 py-1.5 text-sm text-green-700 hover:bg-green-50 rounded-lg transition-colors`
- Position: Leftmost action button (before "设为就绪")
- Color: green-700/green-50 (distinct from blue "set ready" and red "delete" -- green signals "run/start")

**Rationale for green:** The existing `BatchActions` uses blue for "设为就绪" and red for "批量删除". Green provides a clear semantic distinction for "execute/run" and aligns with common start/play button conventions. The Play icon reinforces this association.

**Updated layout:**
```
[已选中 X 项]  [批量执行] [设为就绪] [批量删除]
```

### Tasks Page State Additions

New state variables in `Tasks.tsx`:

```typescript
const [batchExecuteOpen, setBatchExecuteOpen] = useState(false)
const [batchExecuting, setBatchExecuting] = useState(false)

const handleBatchExecute = async (concurrency: number) => {
  setBatchExecuting(true)
  try {
    const result = await batchesApi.create(selectedIds, concurrency)
    toast.success(`已启动 ${selectedIds.length} 个任务的批量执行`)
    setBatchExecuteOpen(false)
    // Phase 73 will navigate to batch progress page
  } catch (error) {
    const message = error instanceof Error ? error.message : '批量执行启动失败'
    toast.error(message)
  } finally {
    setBatchExecuting(false)
  }
}
```
