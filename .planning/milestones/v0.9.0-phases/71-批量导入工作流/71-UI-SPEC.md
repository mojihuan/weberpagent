## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (font-normal) | 1.5 (leading-normal) |
| Table header | 12px (text-xs) | 600 (font-semibold) | 1.5 |
| Heading | 18px (text-lg) | 600 (font-semibold) | 1.2 |
| Page Title | 24px (text-2xl) | 600 (font-semibold) | 1.2 |

**Weight usage rule:** `font-normal` (400) for all non-emphasis text: body copy, cell content, helper text, labels, summary text, counts. `font-semibold` (600) for all emphasis text: titles, page titles, table headers. No `font-medium` (500) anywhere.

**Source:** Pre-populated from existing components. TaskFormModal uses `text-lg font-semibold` for modal title. TaskListHeader uses `text-2xl font-semibold` for page title. StatusBadge and Button previously used `font-medium` -- unified to `font-normal` (body) or `font-semibold` (emphasis) to reduce to 2 weights.

## Component Specifications

### ImportModal (new component)

**Path:** `frontend/src/components/ImportModal/ImportModal.tsx`

**Pattern:** Follows TaskFormModal overlay pattern exactly.

| Property | Value |
|----------|-------|
| Overlay | `fixed inset-0 z-50 flex items-center justify-center` |
| Backdrop | `fixed inset-0 bg-black/50` |
| Container | `relative bg-white rounded-xl shadow-xl w-full max-w-4xl mx-4 max-h-[85vh]` |
| Header | `sticky top-0 bg-white px-6 py-4 border-b border-gray-100 flex items-center justify-between` |
| Title | `text-lg font-semibold text-gray-900` |
| Close button | `text-gray-400 hover:text-gray-600` with lucide X icon (w-5 h-5), `aria-label="关闭"` |
| Body | `p-6 overflow-y-auto` (scrollable area for preview table) |
| Focus | Title bar with step indicator; close button top-right |

**State machine:** Three steps managed by `useState<ImportStep>`:

```
Step 1: "upload"  --> [file selected] --> Step 2: "preview"
Step 2: "preview" --> [confirm click] --> Step 3: "result"
Step 2: "preview" --> [back click]    --> Step 1: "upload"
Step 3: "result"  --> [auto-close after 1.5s] --> onClose + toast + refresh
```

**Props:**

```typescript
interface ImportModalProps {
  open: boolean
  onClose: () => void
  onImportComplete: () => void  // triggers task list refresh
}
```

## API Integration

### Upload API (raw fetch, not apiClient)

```typescript
// frontend/src/api/tasks.ts additions
const API_BASE = import.meta.env.VITE_API_BASE || 'http://localhost:8080/api'

// POST /tasks/import/preview -- FormData, returns ImportPreviewResponse
// POST /tasks/import/confirm -- FormData, returns { status, created_count }
```

**Key constraint:** Use raw `fetch` with `FormData`. Do NOT set Content-Type header (browser auto-sets multipart/form-data with boundary). Do NOT use existing apiClient which hardcodes Content-Type: application/json.

**Source:** CONTEXT.md D-17, RESEARCH.md Pattern 2, STATE.md blocker note.

## Accessibility Notes

- Modal: focus trap via existing pattern (TaskFormModal does not implement focus trap; match existing behavior for consistency)
- Close button: `aria-label="关闭"` on close icon button
- Upload zone: `<input type="file">` is keyboard-accessible, hidden but triggered by zone click
- Preview table: semantic `<table>` with `<thead>`/`<tbody>`, column headers readable by screen readers
- Error rows: text-based error messages (not icon-only), readable by assistive technology
- Confirm disabled state: tooltip or aria-label explaining why disabled ("请修复无效行后再导入")
