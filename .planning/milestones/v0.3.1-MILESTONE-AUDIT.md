## 3. End-to-End Flow Verification

### User Flow: Select Method -> Configure -> Extract -> Use Variable

**Step 1: User selects data method**
- Location: TaskForm.tsx "Get Data" button (line 364-387)
- Action: Opens DataMethodSelector modal
- Backend: GET /api/external-data-methods returns grouped methods

**Step 2: User configures parameters**
- Location: DataMethodSelector Step 2 (line 394-454)
- Action: Fill i, j, k parameter values
- Backend: No call yet (local state only)

**Step 3: User previews data and extracts fields**
- Location: DataMethodSelector Step 3 (line 455-527)
- Action: Click "Preview Data" -> Click on field value
- Backend: POST /api/external-data-methods/execute
- Result: Field path added to extractions

**Step 4: User names variables and confirms**
- Location: DataMethodSelector Step 4 (line 529-594)
- Action: Edit variable names, review code preview
- Result: Code generated in format `imei = context.get_data('BaseParams', 'inventory_list_data', i=2)[0]['imei']`

**Step 5: Code injected into precondition**
- Location: TaskForm.tsx handleDataSelectorConfirm() (line 210-239)
- Action: Code appended to precondition textarea
- Result: Precondition now contains Python code

**Step 6: Execution - context.get_data() called**
- Location: precondition_service.py ContextWrapper.get_data() (line 72-95)
- Action: Precondition code executes, calling context.get_data()
- Result: Data fetched from base_params method, stored in context

**Step 7: Variable substitution in test steps**
- Location: api_assertion_service.py substitute_variables() (line 185-210)
- Action: {{imei}} replaced with actual value
- Result: Test step uses real data value

**E2E Flow Status: VERIFIED**
